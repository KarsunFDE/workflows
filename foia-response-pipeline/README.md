# foia-response-pipeline — project config

Central storage for this project's FDE engine config (Model A).

The engine itself lives at the repo root (`.claude/workflows`, `.claude/skills`) and is installed
into the actual `foia-response-pipeline` project repo, where the workflows are run.

Store here (versioned, group-visible):
- `personas/` cards — `personas/<slug>.md` (+ `personas/README.md` index) generated by `/fde-personas` in the project repo, copied here as the canonical record.
- `fde.config.*` — per-repo context / domain notes / target (if used).
- Sample run outputs (`fde-analysis-report.md`, `fde-modernization-plan.md`) for reference.

> Note: the workflows' Ph0 loader reads `personas/*.md` at the **project root** of wherever they run —
> not this folder. This folder is backup/handoff storage, not an auto-discovery location.
