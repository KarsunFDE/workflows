§1

Executive Summary

FACT The workflow package implements a two-layer LangGraph agent system for FAR-compliant federal contract modification processing, deployed inside the ai-orchestrator FastAPI service.

The outer triage graph (triage_graph.py) ingests contract modifications and invoices, runs multi-agent anomaly detection and FAR-grounded adjudication, and routes each item into one of three lanes: auto_process (policy-cleared, reversible), hitl_escalate (triggers the inner SF-30 graph), or return_route (improper invoice / FAR 32.905). Default-deny: missing data escalates rather than clears.

The inner SF-30 graph (graph.py) assembles a complete SF-30 modification package through a frozen node sequence — lookup, classify, RAG retrieval (ADR-0005), AI draft, CO gate, consent gate, submit — compiled as a subgraph of the triage graph and sharing its MongoDB checkpointer for pause-resume across process restarts.

FACT All authority boundaries are deterministically code-enforced: FAR 43.103 consent derivation (far_rules.py) is pure code, never an LLM call; RESERVED_ACTIONS permanently excludes modification_execution and payment_certification from the auto lane regardless of AI confidence; package_hash (SHA-256) is computed and re-verified at CO gate, consent gate, and submit to detect post-approval mutations; the execution_log uses an atomic insert-before-execute pattern to make the money path replay-safe at the database layer.
What the system does

Automates SF-30 contract modification intake — from SAM.gov contract lookup and block autofill, through FAR-grounded AI drafting, through CO HITL gate, to verified submission to the Java contract-modification-service.

Why it exists

To reduce CO manual workload on routine modifications while satisfying FAR 43.102/43.103, DCAA audit completeness (5+ year retention), and the sponsor mandate that no AI system can authorize payment without a human gate.

Who uses it

Contracting Officers (primary — all irreversible writes), CORs (invoice review and mod initiation), sys_admin (corpus management), DCAA auditors (post-hoc audit replay), OIG reviewers (findings tracker), Vendor PMs (invoice submission, bilateral consent).

Known gaps (Phase 1 debt)

No Pydantic sub-schemas for change_request/invoice dict payloads; live SAM.gov adapter is a documented future swap (mock only); mock_executor.process() is a stub; transactional outbox (D3/Item 2) unimplemented.

Capability Map

Seven distinct capabilities identified. Three are Critical (system inoperable without them). Four are High (significant degradation). All confidence scores are 1.00 (code evidence).
C1 — Contract-of-Record Lookup & SF-30 Field Autofill High 1.00

Deterministic, tenant-scoped SAM.gov contract resolution (no LLM). Populates SF-30 static blocks 2, 3, 6, 7, 8, 10A, 10B, 12 from the award record with source citations. Exposed as both POST /workflow/contract-lookup (UI autofill) and the inner graph lookup_node. Agency isolation enforced at query level (ADR-0006: lookup is not an LLM step).

What breaks if removed: All runs gate to CONTRACT_NOT_FOUND_AWAITING_CO_REVIEW; SF-30 block autofill disappears; no AI substitution permitted.

contract_lookup.py, clients.py (SamGovClient Protocol), nodes_lookup.py, router.py:112-122, sam_gov_client.py

C2 — Multi-Agent Triage Orchestration High 1.00

Outer LangGraph routing modifications and invoices through: anomaly_detector (rule-based flag scanning), adjudicator (LLM-grounded FAR clause verification with prompt-injection hardening), and decision_router (deterministic policy gate). Entry: POST /workflow/triage. Default-deny: missing amount/threshold data escalates. RESERVED_ACTIONS hard-excludes modification_execution and payment_certification from auto lane regardless of confidence (REQ-AGT-2).

What breaks if removed: No intake routing; every item requires manual CO triage; idempotency, anomaly detection, and lane-based audit trail disappear.

triage_graph.py, triage_state.py, nodes_triage.py, auto_approval_policy.py, runner.py:54-64, router.py:125-148

C3 — SF-30 Contract Modification Workflow (HITL Inner Graph) Critical 1.00

Inner LangGraph StateGraph with frozen node order: lookup → validate_lookup → populate_fields → classify → derive_consent → retrieve (RAG) → confidence_check → draft → faithfulness_gate → assemble → co_gate → consent_gate → submit (or supersede on deny). Runs as the hitl_escalate subgraph; inherits parent MongoDB checkpointer.

What breaks if removed: Entire AI-assisted SF-30 capability lost; CO must manually populate all blocks, classify modification type, draft Block 14, and manage consent tracking.

graph.py, state.py, runner.py, nodes_lookup.py, nodes_gate.py

C4 — CO Review Gate & Contract Execution Authority Critical 1.00

HITL interrupt and downstream execution path ensuring only an authenticated, non-anonymous CO can approve and submit. co_gate_node issues a LangGraph interrupt with the package_hash. route_after_co_gate validates strictly against {approved, denied} — any other value routes to terminal supersede. Resume endpoint validates X-User-Id / X-User-Role / X-Tenant-Id headers, cross-checks caller tenant against checkpointed agency_id, and patches verified identity into graph state before resuming.

What breaks if removed: Modifications could be approved by unauthenticated callers; package could be mutated between approval and submission; FAR 43.102 is unenforceable at the AI layer.

nodes_gate.py, router.py:151-251, far_rules.py, modification_client.py, state.py:75-93

C5 — DCAA Audit Trail & Compliance Logging Critical 1.00

Append-only workflow_audit MongoDB collection. Hard frozenset of 11 allowed event_type strings; ValueError on unknown types. High-consequence events (co_decision, contractor_consent_recorded, package_superseded, modification_submitted) require actor_id, actor_role, package_hash in details or record_event raises before writing. Fail-closed: MongoDB write failure re-raises. Every record carries correlation_id threading through all nodes.

What breaks if removed: DCAA audit reconstruction becomes impossible; FAR compliance evidence is lost.

audit_events.py, state.py:75-93, nodes_gate.py, nodes_triage.py

C6 — Money-Path Idempotency & Replay Safety High 1.00

Atomic idempotency ledger (execution_log collection). claim() inserts a processing sentinel before any side effect; unique MongoDB index on idempotency_key makes concurrent race-safety a DB-layer guarantee (DuplicateKeyError returns False). Failed keys locked for operator review — not auto-released, preventing delayed double-pay. auto_process_node refuses to execute without an idempotency_key (fail-closed).

What breaks if removed: Concurrent message redelivery could double-execute money-path auto-process actions.

execution_log.py, nodes_triage.py:221-266, mock_executor.py

C7 — AI Draft Generation & RAG Grounding (Block 14) High 1.00

LLM-backed generation of SF-30 Block 14 modification description text, grounded via the ADR-0005 retrieval pipeline. RetrieveClient Protocol (hybrid search + RRF + cross-encoder rerank, tenant-scoped by agency_id). llm.call_json outputs Pydantic-validated structured output over Bedrock with stub detection. LLM-as-judge confidence scoring with model tier escalation (haiku → sonnet on low confidence). Fail-closed on stub or non-schema-valid LLM output (LLMOutputError routes to CO review).

What breaks if removed: Block 14 must be drafted manually; adjudicator cannot ground verdicts in retrieved clauses.

llm.py, clients.py (RetrieveClient Protocol), state.py (retrieved_chunks, confidence, draft_model_tier fields), nodes_triage.py (adjudicator_node)

Contracting Officer (CO) 1.00 code + CLAUDE.md

Goals: Execute contract modifications lawfully within warrant authority (FAR 1.602-1); obligate funds only after explicit review; maintain a DCAA-replayable audit trail for every high-consequence action.

Responsibilities: Approve/deny every SF-30 mod at co_gate_node; trigger the irreversible submit; enforce FAR 43.103 bilateral consent ordering; certify payment-affecting actions (RESERVED_ACTIONS); curate FAR/DFARS AI corpus within own agency scope.

Pain points: Hard gate on every run with no auto-lane for modification_execution or payment_certification; decision is package_hash-bound so any post-approval package mutation forces full re-approval; identity must be non-anonymous at both orchestrator and Java service layers.

Duplication & Ghosts

Duplications

Duplication MockSamGovClient — dual module-level singletons 0.95

_sam_gov = MockSamGovClient() is instantiated independently in both nodes_lookup.py:26 and router.py:39, creating two separate instances rather than a shared singleton.

INFER For the Phase 1 mock this is harmless — MockSamGovClient has no shared connection state. When the live SAM.gov adapter replaces the mock, two independent instances would duplicate connection pool state and make the swap error-prone.
REC Consolidate to a single shared instance (module-level import from a dedicated deps.py or FastAPI Depends singleton) before the live adapter lands. Low-urgency until Phase 2 adapter work begins.

Ghosts (present on disk, add no capability)

Ghost sam_gov_client.py — pure re-export shim 1.00

Single line: from app.workflow.clients import SamGovClient as SamGovClient. Exists only for import aliasing per m3.md file layout. No business logic, no capability cluster.

Ghost Live SAM.gov Adapter — comment-only reference 1.00

Referenced in clients.py:54-55 docstring as a planned 'client-internal swap' but no live implementation file exists. MockSamGovClient is the only SamGovClient implementor. Acknowledged Phase 1 debt — do not count as a real capability.

Debt mock_executor.process() — stub side-effect recorder 1.00

mock_executor.py:19-23 — process() calls execution_log.mark_processed() which does not exist in the real module (which only has claim / mark_done / mark_failed). The real downstream executor is described only in the module docstring. Phase 1 stub — real capability unimplemented in this package.
